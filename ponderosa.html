1. Environment Setup

Virtual environment

Tested with Python 3.10.9. Python 3.8+ required for phasedibd.

python -m venv ponderosa_venv
source ponderosa_venv/bin/activate
python -m pip install -r requirements.txt
git clone https://github.com/23andMe/phasedibd
cd phasedibd
make
python setup.py install
python tests/unit_tests.py
On some HPCs, replace line 3 of the phasedibd Makefile (pip install cython --user) with python -m pip install cython to avoid permissions issues.

Conda environment

conda env create -f environment.yaml
conda activate ponderosa

Then proceed with steps 4–8 above to install phasedibd.

2. Creating a Map File

Run python pedigree_tools.py -interpolate [options] to create a Plink-formatted MAP file with genetic positions filled in via interpolation from a reference genetic map.

-input_map
MAP file to fill in. No header. Columns: chromosome, SNP ID, cM position (set to 0), physical position.
-genetic_map
Reference genetic map. No header. Requires columns: chromosome, cM position, physical position.
-columns
Optional. 0-indexed column positions for chromosome, cM, and physical position. E.g., -columns 2 3 4. Default assumes Plink MAP format.
-sites
Optional. Subset of loci to output. Two columns (no header): chromosome, physical position. If omitted, all sites in input_map are included.

Output: [input_map]_interpolated.map

3. Running phasedibd

Run python pedigree_tools.py -phasedibd [options] to detect IBD segments.

-input_vcf
Uncompressed VCF for a single chromosome.
-input_map
Plink MAP file. Must contain exactly the sites in input_vcf. Use -interpolate above to create this file.
-outfile
Output file name for IBD segments.

4. Running PONDEROSA

Required file arguments

--ibd
phasedibd IBD output. If all chromosomes are in one file, provide the file name. Otherwise provide chromosome 1 file (assumes "chr1" in the filename).
--fam
PLINK-formatted FAM file.
--king
KING .seg file.

Optional file arguments

--ages
Age file. Column 1: individual ID, column 2: age.
--map
Highly recommended. PLINK-formatted MAP file. All chromosomes can be in one file, or provide chromosome 1.
--populations
For running on a subset of samples. Column 1: individual ID, column 2: population label. Specify target population with --population.
--yaml
Provide all arguments via YAML file instead of the command line.
--pedigree_codes
Instructs PONDEROSA which relationship types to look for.

Other arguments

--output
Output file prefix. Default: Ponderosa.
--min_p
Minimum probability for relationship output. E.g., with min_p=0.95, a pair with P(2nd)=0.98 and P(MHS)=0.93 is reported as 2nd. Default: 0.50.
--population
Used with --populations; specifies which population to run.
--assess
Assess performance on known pairs.
--training
Full path to the degree classifier. Assumes haplotype score (hap) and number of IBD segments (nsegs) classifiers share the same prefix/suffix.
