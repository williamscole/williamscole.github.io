PONDEROSA Simulator

Generates synthetic pedigree data for training and testing PONDEROSA.

1. Installation

The simulator uses a modified version of Ped-Sim that gives the pipeline more control over founder individuals.

git clone https://github.com/williamscole/ped-sim.git
cd ped-sim
make
./setup.sh

Python 3.8 or later is required.

2. Usage & Options

python ponderosa_simulator.py [options]
--num_generations
Number of generations to simulate. Default: 3
--family_size
Average number of children per family. Default: 2
--population_size
Total number of individuals. Default: 100
--output_prefix
Prefix for all output files. Default: simulated_pedigree
--snp_count
Number of SNPs to simulate. Default: 10000
--mutation_rate
Mutation rate per generation. Default: 1e-8
--recombination_rate
Recombination rate. Default: 1e-8
--seed
Random seed for reproducibility. Default: None
--config
Path to a YAML config file for advanced parameter customization.

Example

python ponderosa_simulator.py \
  --num_generations 4 \
  --family_size 3 \
  --population_size 200 \
  --output_prefix my_simulation \
  --snp_count 20000

3. Output Files

[prefix].ped
PLINK-formatted pedigree file.
[prefix].map
PLINK-formatted map file.
[prefix].vcf
VCF file containing simulated genotypes.
[prefix]_relationships.txt
Ground-truth relationships between all pairs of individuals.

5. Integration with PONDEROSA

Simulator output files are compatible with PONDEROSA input formats. Use them to benchmark performance on known pedigree structures or to generate training data for the classifiers.

See the PONDEROSA documentation for details on running the full pipeline.

The simulator is designed for testing and demonstration purposes. While it produces realistic genetic patterns, it may not capture all complexities of real-world inheritance.

6. Troubleshooting

  • Memory issues: For large simulations, reduce population size or SNP count, or run on an HPC environment.
  • Runtime: Simulations with many generations or large populations can be slow. Consider submitting as an HPC batch job.
  • File formats: Ensure PONDEROSA is configured to read the simulator's output formats before running.
